How to Convert a String to an Integer in JavaScript: 8 Methods + Performance Guide

Quick Answer

To convert a string to an integer in JavaScript, use parseInt("123", 10) for strict integer parsing or Number("123") for general number conversion. For the fastest performance, use the unary plus operator: +"123".

Method 1: parseInt() Function (Recommended)

The parseInt() function is the most reliable method for converting strings to integers, especially when you need strict integer values.

Basic Syntax

parseInt(string, radix)

Examples

// Basic usage


const result1 = parseInt("123", 10);        // 123


const result2 = parseInt("123.45", 10);     // 123 (truncates decimal)


const result3 = parseInt("123abc", 10);     // 123 (stops at first non-digit)



// Handling different formats


const result4 = parseInt("010", 10);        // 10 (not 8 in octal)


const result5 = parseInt("0xFF", 16);       // 255 (hexadecimal)


const result6 = parseInt("   42   ", 10);   // 42 (ignores whitespace)

Key Features

  • Always specify radix: Use 10 for decimal numbers
  • Stops at first invalid character: parseInt("123abc", 10) returns 123
  • Handles leading whitespace: Automatically trims spaces
  • Returns NaN for invalid input: parseInt("abc", 10) returns NaN

Best Practices

// ✅ Good - Always specify radix


const num = parseInt(userInput, 10);



// ❌ Avoid - Missing radix (legacy browser issues)


const num = parseInt(userInput);



// ✅ Good - With validation


function parseIntSafe(str) {


    const result = parseInt(str, 10);


    return isNaN(result) ? null : result;


}

Method 2: Number() Function

The Number() function provides strict conversion and is ideal when you need to ensure the entire string represents a valid number.

Examples

// Basic conversion


const num1 = Number("123");           // 123


const num2 = Number("123.45");        // 123.45


const num3 = Number("123abc");        // NaN (strict parsing)



// Special cases


const num4 = Number("");              // 0 (empty string)


const num5 = Number("   ");           // 0 (whitespace only)


const num6 = Number("Infinity");      // Infinity


const num7 = Number("2e3");           // 2000 (scientific notation)

Comparison with parseInt

const input = "123.45abc";



parseInt(input, 10);    // 123 (partial parsing)


Number(input);          // NaN (strict parsing)



// For integers from Number(), use Math functions


const integer = Math.floor(Number("123.45"));  // 123

When to Use Number()

  • Need strict validation (entire string must be a number)
  • Working with decimal numbers
  • Want to handle scientific notation
  • Need consistent behavior across all inputs

Method 3: Unary Plus Operator (Fastest)

The unary plus operator (+) is the fastest method for string-to-number conversion and works identically to Number().

Examples

// Basic usage


const num1 = +"123";           // 123


const num2 = +"123.45";        // 123.45


const num3 = +"123abc";        // NaN



// Performance-critical scenarios


const numbers = ["1", "2", "3", "4", "5"];


const converted = numbers.map(str => +str);  // [1, 2, 3, 4, 5]



// In expressions (be careful with precedence)


const result = +(userInput)  2;

Performance Benefits

// Benchmark results (operations per second)

// Number():       ~650,000,000 ops/sec  


// parseInt():     ~400,000,000 ops/sec

Gotchas to Avoid

function parseToInteger(str, roundingMethod = 'round') {

Method 7: Bitwise Operators

Bitwise operators provide fast integer conversion but have limitations with large numbers.

Double Tilde (~~)

// Double bitwise NOT


const num1 = ~~"123";           // 123


const num2 = ~~"123.45";        // 123 (truncates)


const num3 = ~~"123abc";        // 0 (invalid input becomes 0)



// Equivalent to Math.trunc for valid numbers


const num4 = ~~"-123.45";       // -123

Bitwise OR with Zero

// Bitwise OR with 0


const num1 = "123" | 0;         // 123


const num2 = "123.45" | 0;      // 123


const num3 = "123abc" | 0;      // 0

Zero-fill Right Shift

// Unsigned right shift


const num1 = "123" >>> 0;       // 123


const num2 = "-123" >>> 0;      // 4294967173 (unsigned 32-bit)

⚠️ Important Limitations

// Large number issues (32-bit limitation)


~~"3000000000";                 // -1294967296 ❌


parseInt("3000000000", 10);     // 3000000000 ✅



// Only use for numbers you know are within 32-bit range

const SAFE_MIN = -2147483648;

Common Problems & Solutions

Problem 1: Leading Zeros

// ❌ Problem: Octal interpretation in old browsers


parseInt("010");        // Could return 8 in old browsers



// ✅ Solution: Always specify radix


parseInt("010", 10);    // Always returns 10

Problem 2: Partial Parsing

// ❌ Problem: parseInt accepts partial matches


parseInt("123abc", 10); // Returns 123, ignoring "abc"



// ✅ Solution: Use Number() for strict parsing


function strictParseInt(str) {


    const num = Number(str);


    return Number.isInteger(num) ? num : NaN;


}

Problem 3: Empty Strings and Whitespace

// Different behaviors with empty/whitespace strings


parseInt("", 10);       // NaN


Number("");             // 0


+"";                    // 0


Math.floor("");         // 0



// ✅ Solution: Explicit handling


function safeParseInt(str) {


    if (typeof str !== 'string' || str.trim() === '') {


        return null;


    }


    const result = parseInt(str.trim(), 10);


    return isNaN(result) ? null : result;


}

Problem 4: Large Numbers

// ❌ Problem: Bitwise operators fail with large numbers


~~"9999999999";         // Returns wrong value



// ✅ Solution: Use Number() or parseInt()


parseInt("9999999999", 10);  // Correct result


Number("9999999999");        // Correct result

Problem 5: Scientific Notation

// ❌ Problem: parseInt doesn't understand scientific notation


parseInt("1e3", 10);    // Returns 1, not 1000



// ✅ Solution: Use Number() first


function parseScientific(str) {


    const num = Number(str);


    return Number.isInteger(num) ? num : Math.floor(num);


}



parseScientific("1e3");  // Returns 1000

function validateAndConvertAge(input) {

Frequently Asked Questions

Q: Why does parseInt("010") sometimes return 8?

A: In older JavaScript engines (pre-ES5), parseInt() without a radix parameter would interpret strings starting with "0" as octal (base 8). Always specify the radix: parseInt("010", 10) to ensure decimal interpretation.

// Old behavior (avoid)


parseInt("010");        // Could return 8



// Correct approach


parseInt("010", 10);    // Always returns 10

Q: Which method is fastest for large datasets?

A: The unary plus operator (+str) and multiplication (str 1) are consistently fastest across browsers. However, the difference is only significant when processing millions of conversions.

// Fastest methods


const num1 = +str;      // Unary plus


const num2 = str  1;   // Multiplication


const num3 = str - 0;   // Subtraction

Q: How do I handle NULL and undefined values?

A: Different methods handle these differently. Create a wrapper function for consistent behavior:

function safeConvert(value, defaultValue = 0) {


    if (value == null) return defaultValue;


    const num = Number(value);


    return isNaN(num) ? defaultValue : num;


}



safeConvert(null);      // 0


safeConvert(undefined); // 0


safeConvert("abc");     // 0


safeConvert("123");     // 123

Q: Why does Number("") return 0 instead of NaN?

A: This is JavaScript's defined behavior. Empty strings and whitespace-only strings are converted to 0. If you need stricter validation:

function strictNumber(str) {


    if (typeof str !== 'string' || str.trim() === '') {


        return NaN;


    }


    return Number(str);


}

Q: Can I convert very large numbers safely?

A: Yes, but avoid bitwise operators which are limited to 32-bit integers:

// ✅ Safe for large numbers


const large1 = Number("9007199254740991");      // Max safe integer


const large2 = parseInt("9007199254740991", 10);



// ❌ Unsafe for large numbers


const large3 = ~~"9007199254740991";            // Incorrect result

Q: How do I convert strings with different locales?

A: JavaScript's built-in methods use standard format. For locale-specific parsing:

// For locale-aware parsing, you might need custom logic


function parseLocaleNumber(str, locale = 'en-US') {


    // Remove locale-specific formatting


    if (locale === 'de-DE') {


        // German: 1.234,56 → 1234.56


        str = str.replace(/\./g, '').replace(',', '.');


    }


    return Number(str);


}

Q: What's the difference between parseInt and parseFloat?

A: parseInt stops at the first decimal point and returns an integer, while parseFloat includes decimal places:

parseInt("123.45", 10);    // 123 (integer)


parseFloat("123.45");      // 123.45 (float)



parseInt("123.45abc", 10); // 123


parseFloat("123.45abc");   // 123.45

// Comprehensive validation library


const NumberConverter = {


    toInt(str, options = {}) {


        const { defaultValue = null, allowPartial = false } = options;



        if (typeof str !== 'string') return defaultValue;



        const trimmed = str.trim();


        if (trimmed === '') return defaultValue;

            return Number.isInteger(result) ? result : defaultValue;

    },



    isValidInt(str) {


        return /^-?\d+$/.test(str.trim());


    },



    isValidNumber(str) {


        return !isNaN(Number(str.trim())) && str.trim() !== '';


    }


};
