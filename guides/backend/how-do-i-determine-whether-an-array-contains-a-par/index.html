How do I determine whether an array contains a particular value in Java?

Comprehensive guide to how do i determine whether an array contains a particular value in java?

How to Check if Array Contains Value in Java: 7 Methods + Performance Guide

Quick Answer

To check if an array contains a particular value in Java, use Arrays.asList(array).contains(value) for simplicity, or Arrays.stream(array).anyMatch(value::equals) for Java 8+. For better performance with frequent lookups, convert to a HashSet or use Set.of() in Java 9+.

Method 1: Arrays.asList().contains() (Most Popular)

This is the most commonly used approach, converting the array to a List and using the contains() method.

Basic Usage

public static final String[] VALUES = {"AB", "BC", "CD", "AE"};


String searchValue = "BC";



boolean found = Arrays.asList(VALUES).contains(searchValue);


System.out.println(found); // true

Complete Example

import java.util.Arrays;



public class ArrayContainsExample {


    public static void main(String[] args) {


        String[] fruits = {"apple", "banana", "orange", "grape"};


        String searchFruit = "banana";



        if (Arrays.asList(fruits).contains(searchFruit)) {


            System.out.println("Found " + searchFruit + " in the array!");


        } else {


            System.out.println(searchFruit + " not found in the array.");


        }


    }


}

Pros and Cons

✅ Advantages:

  • Simple and readable
  • Works with all Java versions
  • Handles null values properly
  • No external dependencies

❌ Disadvantages:

  • Creates a lightweight List wrapper (small overhead)
  • Doesn't work with primitive arrays directly
  • O(n) time complexity

Important Notes

⚠️ Warning: This method doesn't work directly with primitive arrays:

int[] numbers = {1, 2, 3, 4};


// This WON'T work as expected - creates List instead of List


boolean found = Arrays.asList(numbers).contains(3); // false!



// Correct approach for primitives - use streams


boolean found = Arrays.stream(numbers).anyMatch(x -> x == 3); // true

Method 3: Manual Loop (Best Performance)

For performance-critical applications, a manual loop is often the fastest approach.

Basic Implementation

public static  boolean contains(T[] array, T value) {


    for (T element : array) {


        if (Objects.equals(element, value)) {


            return true;


        }


    }


    return false;


}

Null-Safe Version

public static  boolean containsNullSafe(T[] array, T value) {


    if (value == null) {


        for (T element : array) {


            if (element == null) {


                return true;


            }


        }


    } else {


        for (T element : array) {


            if (element == value || value.equals(element)) {


                return true;


            }


        }


    }


    return false;


}

Optimized Version with Early Reference Check

public static  boolean containsOptimized(T[] array, T value) {


    for (T element : array) {


        // Check reference equality first (faster)


        if (element == value || (value != null && value.equals(element))) {


            return true;


        }


    }


    return false;


}

Method 6: Binary Search (Sorted Arrays)

For sorted arrays, binary search provides O(log n) performance.

Prerequisites

⚠️ Important: The array MUST be sorted before using binary search!

String[] array = {"AB", "BC", "CD", "AE"}; // Not sorted!


Arrays.sort(array); // Now sorted: ["AB", "AE", "BC", "CD"]



boolean found = Arrays.binarySearch(array, "BC") >= 0; // true

Complete Example

import java.util.Arrays;



public class BinarySearchExample {


    public static void main(String[] args) {


        String[] fruits = {"banana", "apple", "orange", "grape"};



        // Must sort first!


        Arrays.sort(fruits); // ["apple", "banana", "grape", "orange"]



        String searchFruit = "grape";


        int index = Arrays.binarySearch(fruits, searchFruit);



        if (index >= 0) {


            System.out.println("Found " + searchFruit + " at index " + index);


        } else {


            System.out.println(searchFruit + " not found");


        }


    }


}

Example 3: HTTP Status Code Checker

public class HttpStatusChecker {


    private static final int[] SUCCESS_CODES = {200, 201, 202, 204};


    private static final int[] REDIRECT_CODES = {301, 302, 303, 307, 308};


    private static final int[] CLIENT_ERROR_CODES = {400, 401, 403, 404, 409, 422};

        return IntStream.of(SUCCESS_CODES).anyMatch(code -> code == statusCode);

        return IntStream.of(REDIRECT_CODES).anyMatch(code -> code == statusCode);

        if (isRedirect(statusCode)) return "REDIRECT";


        if (isClientError(statusCode)) return "CLIENT_ERROR";


        if (statusCode >= 500) return "SERVER_ERROR";


        return "UNKNOWN";


    }


}

Summary

Key Takeaways

  1. For general use: Arrays.asList(array).contains(value) is the most readable and widely compatible
  2. For modern Java: Arrays.stream(array).anyMatch(value::equals) provides functional programming benefits
  3. For performance: Manual loops are fastest for single lookups; HashSet for multiple lookups
  4. For constants: Use Set.of() in Java 9+ for clean, immutable, and fast lookups
  5. For primitives: Use IntStream.of(array).anyMatch() or similar primitive streams

Decision Matrix

| Scenario | Recommended Method | Reason |

|----------|-------------------|---------|

| One-time lookup, small array | Manual loop | Best performance |

| One-time lookup, readability important | Arrays.asList().contains() | Clean, readable |

| Multiple lookups on same data | HashSet/Set.of() | O(1) lookup time |

| Modern Java codebase | Stream.anyMatch() | Functional style |

| Legacy Java support needed | Arrays.asList().contains() | Universal compatibility |

| Primitive arrays | Primitive streams | Type-safe, efficient |

| Sorted arrays, frequent searches | Binary search | O(log n) performance |

Performance Guidelines

  • Small arrays (< 50 elements): Manual loop or Arrays.asList()
  • Medium arrays (50-1000 elements): Consider usage frequency
  • Large arrays (> 1000 elements): Use HashSet for multiple lookups
  • Frequent searches: Always convert to Set first
  • Memory-constrained: Stick with manual loops

FAQ Section

Q: Why does Arrays.asList().contains() not work with primitive arrays?

A: Java's generics don't support primitive types. When you call Arrays.asList(intArray), it creates a List containing one element (the entire array), not a List. Use streams instead:

int[] numbers = {1, 2, 3};


// Wrong: Arrays.asList(numbers).contains(2) // false


// Correct: Arrays.stream(numbers).anyMatch(x -> x == 2) // true

Q: Which method is fastest for large arrays?

A: It depends on usage:

  • Single lookup: Manual loop (O(n) but minimal overhead)
  • Multiple lookups: HashSet (O(1) after O(n) conversion)
  • Sorted array: Binary search (O(log n))

Q: How do I handle null values safely?

A: Use Objects.equals() for null-safe comparison:

public static  boolean containsNullSafe(T[] array, T value) {


    return Arrays.stream(array).anyMatch(element -> Objects.equals(element, value));


}

Q: Can I use these methods with custom objects?

A: Yes, but ensure your objects properly implement equals() and hashCode():

public class Person {


    private String name;



    @Override


    public boolean equals(Object obj) {


        if (this == obj) return true;


        if (obj == null || getClass() != obj.getClass()) return false;


        Person person = (Person) obj;


        return Objects.equals(name, person.name);


    }



    @Override


    public int hashCode() {


        return Objects.hash(name);


    }


}

Q: What's the difference between HashSet and TreeSet for contains operations?

A:

  • HashSet: O(1) average time, no ordering, allows null
  • TreeSet: O(log n) time, maintains sorted order, no null values
  • Use HashSet for fastest lookups
  • Use TreeSet when you need both fast lookups and sorted iteration

Q: Is it better to use Apache Commons ArrayUtils?

A: Only if you're already using Apache Commons in your project. The built-in Java methods are sufficient for most use cases and don't require external dependencies.

Q: How do I make case-insensitive string comparisons?

A: Use streams with custom predicates:

String[] names = {"John", "jane", "Bob"};


boolean found = Arrays.stream(names)


    .anyMatch("JOHN"::equalsIgnoreCase);

Q: What happens if I modify the original array after using Arrays.asList()?

A: The List returned by Arrays.asList() is backed by the original array, so changes to the array affect the List:

String[] array = {"A", "B", "C"};


List list = Arrays.asList(array);


array[0] = "X";


System.out.println(list); // [X, B, C] - changed!

Q: Can I use binary search on unsorted arrays?

A:* No! Binary search only works on sorted arrays. You must sort first:

String[] array = {"C", "A", "B"};


Arrays.sort(array); // Now: ["A", "B", "C"]


boolean found = Arrays.binarySearch(array, "B") >= 0;

// Copy-paste utility class for your projects


public final class ArrayUtils {



    private ArrayUtils() {} // Utility class



    // Generic contains method


    public static  boolean contains(T[] array, T value) {


        return Arrays.stream(array).anyMatch(element -> Objects.equals(element, value));


    }



    // Null-safe contains


    public static  boolean containsSafe(T[] array, T value) {


        if (array == null) return false;


        return contains(array, value);


    }



    // Case-insensitive string contains


    public static boolean containsIgnoreCase(String[] array, String value) {


        if (array == null || value == null) return false;


        return Arrays.stream(array).anyMatch(value::equalsIgnoreCase);


    }



    // Contains any of the values


    public static  boolean containsAny(T[] array, T... values) {


        if (array == null || values == null) return false;


        Set valueSet = Set.of(values);


        return Arrays.stream(array).anyMatch(valueSet::contains);


    }



    // Primitive array support


    public static boolean contains(int[] array, int value) {


        return Arrays.stream(array).anyMatch(x -> x == value);


    }



    public static boolean contains(double[] array, double value) {


        return Arrays.stream(array).anyMatch(x -> Double.compare(x, value) == 0);


    }



    public static boolean contains(long[] array, long value) {


        return Arrays.stream(array).anyMatch(x -> x == value);


    }


}
