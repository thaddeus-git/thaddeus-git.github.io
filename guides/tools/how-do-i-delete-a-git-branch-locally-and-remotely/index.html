How do I delete a Git branch locally and remotely?

How to Delete a Git Branch Locally and Remotely: Complete Guide + 5 Methods

Quick Answer

To delete a Git branch both locally and remotely, use these commands:

Delete remote branch (Git 1.7.0+)


git push origin --delete 



Delete local branch


git branch -d

For older Git versions, use git push origin : to delete the remote branch.

Ready-to-Use Code

Most Common (Git 1.7.0+) - Copy & Paste Ready


git push origin --delete feature-branch


git branch -d feature-branch



Alternative syntax (same result)


git push -d origin feature-branch


git branch -d feature-branch



For older Git versions (pre-1.7.0)


git push origin :feature-branch


git branch -d feature-branch



Force delete unmerged branch


git branch -D feature-branch


git push origin --delete feature-branch



Clean up stale remote references


git fetch --prune

Method 1: Modern Git Commands (Recommended)

Best for: Git 1.7.0+ users who want clear, readable commands

The modern approach uses explicit --delete flags that make your intentions crystal clear:

Step 1: Delete the remote branch


git push origin --delete 



Step 2: Delete the local branch


git branch -d

Example:

git push origin --delete feature-login


git branch -d feature-login

Why this method is recommended:

  • ✅ Self-documenting (clear what you're doing)
  • ✅ Works with all modern Git versions
  • ✅ Safer than cryptic colon syntax
  • ✅ Automatically removes remote-tracking branches

When to use:

  • You're using Git 1.7.0 or newer
  • You want readable, maintainable scripts
  • You're teaching Git to others

Method 2: Legacy Colon Syntax

Best for: Older Git versions or when you need maximum compatibility

This method uses Git's original push syntax where an empty local branch deletes the remote:

Delete remote branch (legacy syntax)


git push origin :



Delete local branch


git branch -d

Example:

git push origin :feature-login


git branch -d feature-login

How it works:

The colon syntax follows the pattern git push origin [local]:[remote]. When you omit the local part, you're pushing "nothing" to the remote branch, effectively deleting it.

When to use:

  • Git versions older than 1.7.0
  • You're comfortable with Git's advanced syntax
  • Working in environments with mixed Git versions

Method 3: Force Delete Unmerged Branches

Best for: Cleaning up experimental or abandoned branches

Sometimes branches haven't been merged and Git protects you from accidental deletion:

Force delete local branch (even if unmerged)


git branch -D 



Delete remote branch


git push origin --delete

Example:

git branch -D experimental-feature


git push origin --delete experimental-feature

⚠️ Warning: The -D flag bypasses Git's safety checks. Use only when you're certain the branch can be discarded.

When to use:

  • Abandoning experimental work
  • Cleaning up feature branches that won't be merged
  • You've already merged changes manually

Method 4: GitHub Desktop GUI

Best for: Visual learners and GUI-preferred workflows

Using GitHub Desktop application:

  1. Launch GitHub Desktop
  2. Select your repository
  3. Switch to the branch you want to delete
  4. From Branch menu → Select "Unpublish..." (deletes remote)
  5. From Branch menu → Select "Delete [branch-name]..." (deletes local)

When to use:

  • You prefer graphical interfaces
  • You're new to Git command line
  • You want visual confirmation before deletion

Method 5: Batch Delete with Aliases

Best for: Power users who frequently delete branches

Create a Git alias to delete both local and remote branches in one command:

Set up the alias


git config --global alias.rmbranch '!f(){ git branch -d ${1} && git push origin --delete ${1}; };f'



Usage


git rmbranch feature-branch

Advanced alias with error handling:

git config --global alias.kill '!f(){ git branch -D "$1"; git push origin --delete "$1"; };f'

Bash function alternative:

Add to your ~/.bashrc or ~/.zshrc


function gitdelete(){


    git push origin --delete $1


    git branch -D $1


}



Usage


gitdelete feature-branch

Understanding 3 Types of Branches

Critical concept: When deleting Git branches, you're actually dealing with three different branches:

🏠 Local Repository          ☁️ Remote Repository (origin)


├── 1. Local branch          ├── 2. Remote branch


│   └── feature-login        │   └── feature-login


└── 3. Remote-tracking       


    └── origin/feature-login

1. Local Branch

  • Location: Your local machine
  • Purpose: Where you make commits
  • Delete with: git branch -d

2. Remote Branch

  • Location: Remote server (GitHub, GitLab, etc.)
  • Purpose: Shared with team members
  • Delete with: git push origin --delete

3. Local Remote-Tracking Branch

  • Location: Your local machine (cached copy)
  • Purpose: Tracks the state of remote branches
  • Delete with: Automatic when you delete remote, or git fetch --prune

Why this matters: Deleting only the local branch leaves the remote branch intact. Deleting only the remote branch leaves stale references locally.

Common Problems & Solutions

Problem 1: "error: branch 'remotes/origin/branch' not found"

❌ Wrong


git branch -d remotes/origin/feature



✅ Correct


git push origin --delete feature


git branch -d feature

Solution: Don't try to delete remote-tracking branches directly. Delete the actual remote branch instead.

Problem 2: "error: The branch 'feature' is not fully merged"

Error message


error: The branch 'feature' is not fully merged.


If you are sure you want to delete it, run 'git branch -D feature'.



✅ Solutions


Option 1: Force delete (if you're sure)


git branch -D feature



Option 2: Merge first, then delete


git checkout main


git merge feature


git branch -d feature

Problem 3: "error: dst refspec matches more than one"

This happens when you have a branch and tag with the same name:

❌ Ambiguous


git push origin :release-1.0



✅ Specific


git push origin :refs/heads/release-1.0  # Delete branch


git push origin :refs/tags/release-1.0   # Delete tag

Problem 4: Remote branch still appears after deletion

Other team members need to clean up stale references:

Clean up obsolete remote-tracking branches


git fetch --prune


or


git remote prune origin

Problem 5: Can't delete currently checked out branch

❌ Error if you're on the branch


git branch -d current-branch



✅ Switch first


git checkout main


git branch -d current-branch

List merged branches

git fetch --prune

Summary

Key Takeaways

  1. Use modern syntax: git push origin --delete is clearer than git push origin :
  1. Remember the three branches: Local, remote, and remote-tracking branches are all separate entities
  1. Safety first: Use -d instead of -D unless you're certain about force deletion
  1. Clean up regularly: Run git fetch --prune to remove stale remote references
  1. Check before deleting: Use git branch --merged to see which branches are safe to delete

Frequently Asked Questions

Q: Why does my query return "Everything up-to-date" after deleting?

A: You likely deleted the local remote-tracking branch instead of the actual remote branch. Use git push origin --delete to delete the remote branch properly.

Q: Which method is fastest for large repositories?

A: All deletion methods are equally fast since they only remove references, not commit data. The speed difference is negligible. Choose based on Git version compatibility and readability preferences.

Q: How do I handle NULL values or non-existent branches?

A: Git will show an error if you try to delete a non-existent branch:

Check if branch exists first


git show-ref --verify --quiet refs/heads/ && git branch -d 



For remote branches


git ls-remote --exit-code origin  && git push origin --delete

Q: Can I undo a branch deletion?

A: Yes, if you know the commit hash:

Find the last commit of the deleted branch


git reflog



Recreate the branch


git checkout -b recovered-branch

Q: What's the difference between -d and -D flags?

A:

  • -d (safe delete): Only deletes if the branch has been merged
  • -D (force delete): Deletes regardless of merge status
  • Use -d by default for safety

Q: Why do I still see the branch after deletion?

A: You're likely seeing a stale remote-tracking branch. Clean it up with:

git fetch --prune

Q: How do I delete branches with special characters in names?

A: Quote the branch name:

git branch -d "feature/user-auth"


git push origin --delete "feature/user-auth"

Q: Can I delete the main/master branch?

A: Most Git hosting services protect the default branch. You'll get an error like "cannot delete main branch." You'd need to change the default branch first in your repository settings.

Q: What happens to pull requests when I delete a branch?

A:

  • GitHub/GitLab: Pull requests remain but show as "closed"
  • Local changes: Any unmerged commits in the branch are lost
  • Best practice: Merge or close pull requests before deleting branches

Q: How do I delete all merged branches at once?

A: Use this command (be careful!):

git branch --merged main | grep -v "main\|develop" | xargs -n 1 git branch -d

This finds all branches merged into main, excludes main and develop, then deletes the rest.
